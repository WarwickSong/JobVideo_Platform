# 常见问题

## 1. 域名打不开

常见原因：

- 域名还没生效
- DNS 解析错了
- 安全组没放行 `80`/`443`
- Nginx 没启动

可以检查：

```bash
sudo systemctl status nginx
sudo nginx -t
```

## 2. 首页能打开，但视频加载失败

常见原因：

- `VIDEO_STORAGE_DIR` 配置错了
- Nginx `/videos/` 的 `alias` 配置错了
- 视频文件没有真正上传到服务器
- 视频文件目录权限不正确

当前推荐配置：

```env
VIDEO_STORAGE_DIR=/var/www/zhenzhao.top/video_storage
```

Nginx 应配置：

```nginx
location /videos/ {
    alias /var/www/zhenzhao.top/video_storage/;
}
```

检查：

```bash
ls -lah /var/www/zhenzhao.top/video_storage
curl -I https://zhenzhao.top/videos/某个视频文件名.mp4
```

## 3. 页面打开了，但视频流接口报错

常见原因：

- 后端服务没启动
- Nginx `/api/` 转发错了
- 后端 `.env` 没配置好
- 数据库或表结构异常

可以先检查：

```bash
sudo systemctl status zhenzhao-backend
sudo journalctl -u zhenzhao-backend -n 100 --no-pager
curl http://127.0.0.1:8000/video/feed
curl https://zhenzhao.top/api/video/feed
```

Nginx `/api/` 应这样配置：

```nginx
location /api/ {
    proxy_pass http://127.0.0.1:8000/;
}
```

`proxy_pass` 末尾的 `/` 很重要。没有这个 `/` 时，`/api/video/feed` 可能不会转成后端真实的 `/video/feed`。

## 4. 后端服务启动失败

检查日志：

```bash
sudo journalctl -u zhenzhao-backend -n 100 --no-pager
```

重点检查：

- `WorkingDirectory=/var/www/zhenzhao.top/app/jobvideo_backend`
- `EnvironmentFile=/var/www/zhenzhao.top/app/jobvideo_backend/.env`
- `ExecStart=/var/www/zhenzhao.top/app/jobvideo_backend/.venv/bin/uvicorn app.main:app --host 127.0.0.1 --port 8000`
- `.venv` 是否存在
- `requirements.txt` 是否安装成功

## 5. HTTPS 申请失败

常见原因：

- 域名还没解析到当前服务器
- `80` 端口没通
- Nginx 配置错误
- 云服务商安全组没有放行 `80`/`443`

检查：

```bash
sudo nginx -t
curl -I http://zhenzhao.top
```

## 6. 修改了前端代码，但线上页面没变化

你可能只是改了源代码，没有重新构建并复制到 Nginx 静态目录。

需要重新执行：

```bash
cd /var/www/zhenzhao.top/app/jobvideo_frontend
npm ci
npm run build
rm -rf /var/www/zhenzhao.top/frontend-dist/*
cp -r dist/* /var/www/zhenzhao.top/frontend-dist/
sudo systemctl reload nginx
```

## 7. 上传视频失败

常见原因：

- Nginx 请求体限制太小
- 视频目录没有写入权限
- 后端 `.env` 的 `VIDEO_STORAGE_DIR` 配置错误

Nginx 应包含：

```nginx
client_max_body_size 200m;
```

目录权限检查：

```bash
sudo chown -R ubuntu:ubuntu /var/www/zhenzhao.top/video_storage
sudo chmod -R 755 /var/www/zhenzhao.top/video_storage
```

如果 systemd 使用的不是 `ubuntu` 用户，把命令里的用户改成实际用户。

## 8. 演示登录失败

当前前端显示演示登录面板，后端数据库里必须存在对应账号。

进入后端目录执行：

```bash
cd /var/www/zhenzhao.top/app/jobvideo_backend
source .venv/bin/activate
python create_demo_users.py
sudo systemctl restart zhenzhao-backend
```

如果这是正式生产站点，不建议长期公开演示账号密码。
