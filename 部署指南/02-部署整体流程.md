# 部署整体流程

整件事可以理解成 8 步。

## 第 1 步：准备服务器和域名

准备一台能访问互联网的 Ubuntu 24.04 LTS 云服务器，以及一个域名。

## 第 2 步：把域名指向服务器

让域名解析到服务器公网 IP。

当前目标信息：

- 域名：`zhenzhao.top`
- 服务器 IP：`124.220.216.17`

## 第 3 步：初始化服务器环境

安装 Git、Nginx、Python、Node.js 22 LTS、Certbot 等基础软件。

后端环境使用 Python 自带 `venv`，不需要安装 conda。

## 第 4 步：把项目上传到服务器

推荐把整个 `JobVideo_Platform` 作为一个仓库放到：

```text
/var/www/zhenzhao.top/app/
```

最终结构应类似：

```text
/var/www/zhenzhao.top/
  app/
    jobvideo_backend/
    jobvideo_frontend/
  frontend-dist/
  video_storage/
  logs/
```

## 第 5 步：启动后端

进入：

```bash
cd /var/www/zhenzhao.top/app/jobvideo_backend
```

创建 `.venv`，安装依赖，配置 `.env`，然后用 systemd 运行：

```bash
uvicorn app.main:app --host 127.0.0.1 --port 8000
```

后端只监听本机 `127.0.0.1:8000`，不直接暴露给外网。

## 第 6 步：构建前端

进入：

```bash
cd /var/www/zhenzhao.top/app/jobvideo_frontend
```

创建 `.env.production`：

```env
VITE_API_BASE_URL=/api
```

然后执行：

```bash
npm ci
npm run build
```

构建结果复制到：

```text
/var/www/zhenzhao.top/frontend-dist/
```

## 第 7 步：配置 Nginx

让 Nginx：

- 对外提供前端页面
- 把 `/api/` 转发到后端，并去掉 `/api` 前缀
- 把 `/videos/` 映射到视频目录
- 限制或放宽上传大小，例如 `client_max_body_size 200m`

## 第 8 步：申请 HTTPS 并检查上线

使用 Certbot 申请 HTTPS：

```bash
sudo certbot --nginx -d zhenzhao.top -d www.zhenzhao.top
```

上线后检查：

- `https://zhenzhao.top`
- `https://zhenzhao.top/api/video/feed`
- `https://zhenzhao.top/videos/...`

## 需要反复记住的一点

外部用户最终只访问你的域名，不直接访问 `8000` 或 `5173` 端口。

也就是说：

- 开发环境：前端和后端各跑各的
- 生产环境：用户只访问 Nginx 提供的统一入口
