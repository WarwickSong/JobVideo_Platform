# 公网部署总指南

这份指南是给第一次做部署的人准备的。

目标是把当前项目部署到一台公网 Linux 服务器上，让别人通过域名访问：

- 前端页面：`https://你的域名`
- 后端接口：`https://你的域名/api/...`
- 视频资源：`https://你的域名/videos/...`

这套方案采用最稳妥、最容易理解的一种结构：

- 一台 Linux 云服务器
- `Nginx` 负责对外访问
- `FastAPI + uvicorn` 运行后端
- `Vite build` 后的静态文件由 `Nginx` 提供
- 视频文件仍保存在服务器本地目录
- 第一版先继续用 `SQLite`

为什么先这样做：

- 成本低
- 结构简单
- 对新手最容易落地
- 先把系统跑起来，再考虑 PostgreSQL、对象存储、Docker、CI/CD
