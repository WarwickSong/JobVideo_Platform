# 公网部署总指南

这份指南是给第一次做部署的人准备的，目标是把当前 `JobVideo_Platform` 部署到一台公网 Linux 云服务器上，让别人通过域名访问：

一般方案：
- 前端页面：`https://你的域名`
- 后端接口：`https://你的域名/api/...`
- 视频资源：`https://你的域名/videos/...`

当前实际输入：
- 前端页面：`https://zhenzhao.top`
- 后端接口：`https://zhenzhao.top/api/...`
- 视频资源：`https://zhenzhao.top/videos/...`

当前推荐以这份完整指南为准：

- [云服务器部署/JobVideo_Platform 云服务器部署指南.md](./云服务器部署/JobVideo_Platform%20云服务器部署指南.md)

根目录下的 01 到 10 是拆分版说明，适合分步骤阅读。它们应与完整指南保持一致。

## 当前部署结构

这套方案采用最稳妥、最容易理解的一种结构：

- 一台 Ubuntu 24.04 LTS 云服务器
- `Nginx` 负责对外访问、HTTPS、前端静态文件和反向代理
- `FastAPI + uvicorn` 运行后端
- 前端使用 Vue 3 + Vite 7，构建后的静态文件由 `Nginx` 提供
- 视频文件保存在服务器本地目录 `/var/www/zhenzhao.top/video_storage`
- 第一版先继续使用 `SQLite`

为什么先这样做：

- 成本低
- 结构简单
- 对新手最容易落地
- 先把系统跑起来，再考虑 PostgreSQL、对象存储、Docker、CI/CD

## 服务器上的关键路径

```text
/var/www/zhenzhao.top/
  app/
    jobvideo_backend/
    jobvideo_frontend/
  frontend-dist/
  video_storage/
  logs/
```

含义：

- `app/`：完整项目源码
- `app/jobvideo_backend/`：FastAPI 后端
- `app/jobvideo_frontend/`：Vue 3 + Vite 前端
- `frontend-dist/`：前端构建后的静态文件
- `video_storage/`：视频文件目录

## 你和我分别负责什么

我已经帮你整理这些内容：

- 按当前项目实际结构修订部署路径
- 明确后端用 `uvicorn app.main:app` 启动
- 明确前端用 Vite 构建
- 明确 Nginx `/api/` 和 `/videos/` 的配置方式

你需要亲手完成的内容：

- 购买服务器
- 购买域名并解析到服务器 IP
- 登录服务器执行命令
- 上传或拉取当前项目代码
- 配置 `.env`
- 配置 Nginx 和 HTTPS
- 检查上线结果

## 当前代码状态说明

当前前端已经有演示账号登录面板，后端也提供了创建演示账号的脚本。

这意味着：

- 公开视频流可以先上线
- 如果要使用演示登录，需要在后端执行 `python create_demo_users.py` 创建演示账号
- 生产环境仍然应设置 `ENABLE_TEST_TOKEN=false`
- 如果未来不是演示站，而是真实用户系统，应补充完整注册、登录、权限和账号管理流程
