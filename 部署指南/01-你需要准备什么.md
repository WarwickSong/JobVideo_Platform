# 你需要准备什么

部署前，你需要准备以下东西。

## 必须有

1. 一台 Linux 云服务器
2. 一个域名
3. 能登录服务器的账号和密码，或者 SSH 密钥
4. 这个项目的完整代码
5. 当前项目需要展示或上传的视频文件

## 推荐服务器配置

第一版建议：

- 操作系统：Ubuntu 24.04 LTS
- CPU：2 核
- 内存：2GB 或以上
- 磁盘：40GB 或以上
- 带宽：按你的预算和视频访问量选择

如果只是自己演示给别人看，这个配置通常够用。

## 服务器上需要的软件

服务器上需要：

- Git
- Nginx
- Python 3 + `venv`
- Node.js 22 LTS
- npm
- Certbot

服务器上不需要：

- conda
- Vue CLI
- Flask
- Django
- Gunicorn
- Docker

当前项目后端依赖较轻，生产部署用 Python 自带的 `.venv` 更简单稳定。

## 你的本地电脑上最好也准备好

- Git
- Node.js 22 LTS
- Python 3.11 或 Python 3.12
- 一个能正常连接服务器的终端工具

Windows 常见工具：

- `PowerShell`
- `Windows Terminal`
- `PuTTY`
- `MobaXterm`

如果你本地已经用 conda 开发，可以继续用 conda 做本地验证；但云服务器部署不建议额外安装 conda。

## 你现在不需要理解的东西

第一次部署时，不需要先搞懂这些概念：

- Kubernetes
- CI/CD
- 对象存储
- 负载均衡
- 容器编排

先把单机版本上线，再逐步升级。
