# 8080 端口临时测试方案与正常方案切换清单

本文用于记录 `zhenzhao.top` 在腾讯云中国境内服务器上的访问方案切换。需要特别注意：`8080` 不能绕过未备案域名拦截。未备案时，`http://zhenzhao.top:8080` 仍然可能提示“网站未完成备案”。

## 结论

### 不能作为备案替代的方案

以下地址在腾讯云中国境内服务器上可能都会因为域名未备案而被拦截：

```text
http://zhenzhao.top
http://zhenzhao.top:8080
https://zhenzhao.top
```

### 可以作为临时测试的方案

未备案阶段，优先使用公网 IP + 8080 做临时功能测试：

```text
http://124.220.216.17:8080
http://124.220.216.17:8080/api/video/feed
```

### 正式上线推荐方案

正式上线建议完成备案后使用：

```text
https://zhenzhao.top
https://zhenzhao.top/api/video/feed
```

## 端口说明

- `80`：HTTP 默认端口，对应 `http://zhenzhao.top`
- `443`：HTTPS 默认端口，对应 `https://zhenzhao.top`
- `8000`：后端内部端口，只监听 `127.0.0.1:8000`，不直接开放公网
- `8080`：临时 HTTP 测试端口，建议配合公网 IP 使用

## 当前服务器已做的变动

当前已把 Nginx 站点配置切换为 8080：

```nginx
server {
    listen 8080;
    server_name zhenzhao.top www.zhenzhao.top;
}
```

配置文件位置：

```bash
/etc/nginx/sites-available/zhenzhao.top
```

当前推荐测试地址：

```text
http://124.220.216.17:8080
```

## 从正常 HTTP 方案切换到 IP + 8080 临时测试方案

### 1. 修改 Nginx 监听端口

编辑站点配置：

```bash
sudo vim /etc/nginx/sites-available/zhenzhao.top
```

把：

```nginx
listen 80;
```

改成：

```nginx
listen 8080;
```

### 2. 检查并重载 Nginx

```bash
sudo nginx -t
sudo systemctl reload nginx
```

### 3. 放行 8080 端口

如果服务器启用了 `ufw`：

```bash
sudo ufw allow 8080/tcp
sudo ufw status
```

还需要到腾讯云控制台的安全组中放行：

```text
TCP 8080
来源 0.0.0.0/0
```

### 4. 检查监听端口

```bash
sudo ss -tlnp | grep -E '(:8080|:8000)'
```

应该能看到：

```text
0.0.0.0:8080
127.0.0.1:8000
```

### 5. 检查访问

服务器本机检查：

```bash
curl http://127.0.0.1:8000/
curl http://127.0.0.1:8000/video/feed
curl http://127.0.0.1:8080/
curl http://127.0.0.1:8080/api/video/feed
```

公网 IP 检查：

```bash
curl http://124.220.216.17:8080/
curl http://124.220.216.17:8080/api/video/feed
```

浏览器访问：

```text
http://124.220.216.17:8080
```

## 从 IP + 8080 临时测试方案切回正常 HTTP 方案

### 1. 修改 Nginx 监听端口

编辑站点配置：

```bash
sudo vim /etc/nginx/sites-available/zhenzhao.top
```

把：

```nginx
listen 8080;
```

改回：

```nginx
listen 80;
```

### 2. 检查并重载 Nginx

```bash
sudo nginx -t
sudo systemctl reload nginx
```

### 3. 检查监听端口

```bash
sudo ss -tlnp | grep -E '(:80|:8000)'
```

应该能看到：

```text
0.0.0.0:80
127.0.0.1:8000
```

### 4. 检查访问

```bash
curl http://127.0.0.1:8000/
curl http://127.0.0.1:8000/video/feed
curl http://zhenzhao.top/
curl http://zhenzhao.top/api/video/feed
```

浏览器访问：

```text
http://zhenzhao.top
```

如果仍然提示“网站未完成备案”，说明域名访问被腾讯云备案策略拦截，需要完成备案后再使用域名访问。

## 从 HTTP 正常方案升级到 HTTPS 正常方案

### 1. 确认前提

需要满足：

- 域名 DNS 已解析到服务器公网 IP
- 腾讯云备案已完成并接入当前服务器资源
- 云服务器安全组已放行 `80` 和 `443`
- Nginx 正常监听 `80`
- 服务器可以通过域名正常访问 HTTP

### 2. 申请 HTTPS 证书

```bash
sudo certbot --nginx -d zhenzhao.top -d www.zhenzhao.top
sudo certbot renew --dry-run
```

### 3. 检查监听端口

```bash
sudo ss -tlnp | grep -E '(:80|:443|:8000)'
```

应该能看到：

```text
0.0.0.0:80
0.0.0.0:443
127.0.0.1:8000
```

### 4. 检查访问

```bash
curl http://127.0.0.1:8000/
curl http://127.0.0.1:8000/video/feed
curl https://zhenzhao.top/
curl https://zhenzhao.top/api/video/feed
```

浏览器访问：

```text
https://zhenzhao.top
```

## 前端配置是否需要修改

这些方案下，前端生产环境配置都保持不变：

```env
VITE_API_BASE_URL=/api
```

原因是前端页面和接口请求保持同源：

- IP + 8080 临时测试方案：页面 `http://124.220.216.17:8080`，接口 `/api` 会请求 `http://124.220.216.17:8080/api`
- HTTP 正常方案：页面 `http://zhenzhao.top`，接口 `/api` 会请求 `http://zhenzhao.top/api`
- HTTPS 正常方案：页面 `https://zhenzhao.top`，接口 `/api` 会请求 `https://zhenzhao.top/api`

Nginx 会统一把 `/api/` 转发到后端：

```nginx
location /api/ {
    proxy_pass http://127.0.0.1:8000/;
}
```

## 后端配置是否需要修改

不需要。

后端 systemd 服务继续监听本机内部地址：

```text
127.0.0.1:8000
```

不要把后端直接改成公网监听，也不建议直接开放 `8000` 到公网。

## 视频访问是否需要修改

不需要。

视频地址仍然使用：

```text
/videos/文件名.mp4
```

Nginx 会通过以下配置映射到服务器视频目录：

```nginx
location /videos/ {
    alias /var/www/zhenzhao.top/video_storage/;
}
```

## 可选正式解决方案

如果需要让 `zhenzhao.top` 域名正常访问，有以下选择：

- 在腾讯云完成备案，并确保备案接入当前腾讯云资源
- 使用已经备案并接入腾讯云的域名
- 把网站迁移到中国境外服务器，再使用域名解析到境外服务器
- 暂时不用域名，只用 `http://124.220.216.17:8080` 做功能测试

## 快速切换命令

如果要切回 80：

```bash
sudo perl -0pi -e 's/listen 8080;/listen 80;/' /etc/nginx/sites-available/zhenzhao.top
sudo nginx -t
sudo systemctl reload nginx
```

如果要再次切到 8080：

```bash
sudo perl -0pi -e 's/listen 80;/listen 8080;/' /etc/nginx/sites-available/zhenzhao.top
sudo nginx -t
sudo systemctl reload nginx
```
