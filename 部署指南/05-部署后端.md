# 部署后端

## 目标

让后端在服务器后台稳定运行。

## 1. 进入后端目录

```bash
cd /var/www/jobvideo/backend/jobvideo_backend
```

注意：

- 如果你上传后的目录结构不一样，这里要按你的实际路径调整

## 2. 创建虚拟环境

```bash
python3 -m venv .venv
source .venv/bin/activate
```

## 3. 安装依赖

```bash
pip install -r requirements.txt
```

## 4. 创建 `.env`

你可以先参考仓库里的：

- [jobvideo_backend/.env.example](/C:/Coding/JobVideo_Platform/jobvideo_backend/.env.example)

在服务器里新建 `.env`：

```env
APP_ENV=production
SECRET_KEY=换成你自己的长随机字符串
ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=60
DATABASE_URL=sqlite:///./test.db
VIDEO_STORAGE_DIR=/var/www/jobvideo/video_storage
FRONTEND_ORIGINS=https://你的域名
ENABLE_TEST_TOKEN=false
```

## 5. 先手动测试启动

```bash
uvicorn app.main:app --host 127.0.0.1 --port 8000
```

说明：

- 这里先监听 `127.0.0.1`
- 因为生产环境由 Nginx 代理访问，不需要直接暴露 `8000`

## 6. 测试接口

新开一个终端窗口，执行：

```bash
curl http://127.0.0.1:8000/
```

如果返回 JSON，说明后端基本启动成功。

## 7. 创建 systemd 服务

创建文件：

```text
/etc/systemd/system/jobvideo-backend.service
```

内容如下：

```ini
[Unit]
Description=JobVideo FastAPI Backend
After=network.target

[Service]
User=www-data
Group=www-data
WorkingDirectory=/var/www/jobvideo/backend/jobvideo_backend
EnvironmentFile=/var/www/jobvideo/backend/jobvideo_backend/.env
ExecStart=/var/www/jobvideo/backend/jobvideo_backend/.venv/bin/uvicorn app.main:app --host 127.0.0.1 --port 8000
Restart=always

[Install]
WantedBy=multi-user.target
```

## 8. 启动服务

```bash
sudo systemctl daemon-reload
sudo systemctl enable jobvideo-backend
sudo systemctl start jobvideo-backend
sudo systemctl status jobvideo-backend
```

## 9. 如果启动失败

查看日志：

```bash
sudo journalctl -u jobvideo-backend -n 100 --no-pager
```
