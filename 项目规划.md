# 基于手机瀑布流短视频的招聘平台软件

这是一个**以「短视频」为核心媒介的招聘平台**，本质是：

> **视频流 × 招聘 × 双角色用户 × 可扩展内容绑定系统**

---

## 1️⃣ 客户端（求职者端 / 普通用户端）

> 核心目标：**刷视频 → 产生兴趣 → 行动（投递 / 收藏 / 联系）**

### 🔹 核心模块

1. **瀑布流 / Feed 流**

   * 类似抖音 / 小红书
   * 视频类型混合：

     * 职位视频
     * 简历视频
     * 公司介绍视频
     * （未来）培训 / 宣讲 / 项目展示

2. **视频详情页**

   * 视频播放
   * 绑定对象展示（职位 / 简历 / 公司等）
   * 操作：

     * 收藏
     * 点赞
     * 投递（如果是职位）
     * 联系 / 关注

3. **我的简历**

   * 文本简历
   * 绑定一个或多个「简历视频」
   * 不同阶段可绑定不同视频（这是你现在设计得很好的地方）

4. **个人中心**

   * 基本信息
   * 投递记录
   * 收藏视频
   * 上传的视频列表

---

## 2️⃣ 企业端（HR / 招聘方）

> 核心目标：**发布岗位 → 用视频表达 → 筛选候选人**

### 🔹 核心模块

1. **企业信息**

   * 公司介绍（可绑定视频）
   * 企业主页（视频流）

2. **职位管理**

   * 创建 JD（文本）
   * 绑定职位视频（核心差异点）
   * 职位状态管理（开放 / 关闭）

3. **候选人管理**

   * 投递列表
   * 简历视频查看
   * 初步筛选（未来可引入 AI）

4. **数据看板（未来）**

   * 视频曝光量
   * 投递转化率
   * 候选人行为

---

## 3️⃣ 后端系统

> 核心原则：**结构清晰、角色解耦、视频绑定可扩展**

### 🔹 当前采用的架构思想

* **FastAPI**
* **模块化目录结构**
* **REST API**
* **统一用户系统（user + role）**
* **Video 是核心实体**
* **Video → Target 的通用绑定机制**

### 🔹 关键设计

```text
Video
 ├── target_type  (job / resume / company_intro / future)
 └── target_id
```

👉 这意味着：

* 视频是“一等公民”
* 绑定对象是**可扩展插件**
* 不会被 JD 或简历「绑死」

### ▶️ 下一步推荐顺序（不是全做，是阶段性）
