# 局域网手机访问说明

这份文档是针对**开发者**的，说明如何在本地开发时，让同一个局域网下的手机访问到正在运行的前后端页面。

## 适用场景

- 你在电脑上启动了前后端服务
- 想用手机（iPhone / Android）在同一 Wi-Fi 下打开页面，测试真实体验
- 不需要部署到服务器

## 一、准备工作

在操作之前，请确认：

1. 电脑和手机连接的是**同一个 Wi-Fi**（或同一个局域网）
2. 电脑上的后端服务和前端服务都已经启动
3. 电脑的防火墙没有阻止 5173 和 8000 端口

## 二、查看电脑的局域网 IP

Windows 系统：

1. 打开命令提示符（Win + R → 输入 `cmd` → 回车）
2. 输入以下命令后回车：
   ```powershell
   ipconfig
   ```
3. 找到你的网络适配器，查看 **IPv4 地址**，格式类似：
   ```
   192.168.x.x
   192.168.31.184
   ```
   把这个地址记下来，后面会用到。

## 三、启动前后端服务

如果还没启动，按以下步骤：

### 启动后端（先启动）

在 `jobvideo_backend` 目录下运行：

```powershell
cd C:\Coding\JobVideo_Platform\jobvideo_backend
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

`--host 0.0.0.0` 让后端监听所有网络请求（包括来自局域网手机的请求）。

### 启动前端（后启动）

在 `jobvideo_frontend` 目录下运行：

```powershell
cd C:\Coding\JobVideo_Platform\jobvideo_frontend
npm run dev
```

前端已经配置了 `host: true`（在 vite.config.js 中），Vite 会自动监听 `0.0.0.0`，手机可以访问。

## 四、在手机上打开

用手机浏览器访问以下地址：

```text
http://192.168.x.x:5173
http://192.168.31.184:5173
```

把 `192.168.x.x` 替换成第二步查到的电脑 IP。

页面打开后，视频、登录、点赞、收藏等所有功能都应该正常工作。

## 五、如果打不开怎么办

### 1. 防火墙阻止了访问

Windows 可能会弹出防火墙提示，选择"允许访问"。如果之前点了阻止，可手动添加规则：

1. 打开 **控制面板 → Windows Defender 防火墙 → 高级设置**
2. 点击 **入站规则 → 新建规则**
3. 规则类型选 **端口**
4. 端口填 `5173, 8000`（前后端各一个端口）
5. 选择 **允许连接**
6. 按提示完成即可

### 2. 前端报错 "无法连接后端"

在手机浏览器中打开开发者工具（部分浏览器支持），查看网络请求是否报错。

常见原因：

- 后端没有加 `--host 0.0.0.0`
- 前端 `.env` 文件中 `VITE_DEV_BACKEND_URL` 写的是 `localhost`，应改成电脑的局域网 IP，例如：
  ```
  VITE_DEV_BACKEND_URL=http://192.168.x.x:8000
  VITE_DEV_BACKEND_URL=http://192.168.31.184:8000
  ```
  改完后重启前端服务。

### 3. 手机和电脑不在同一网段

有些公司或公共场所的 Wi-Fi 会隔离设备（即使用同一 Wi-Fi，设备之间也不能互通）。这种情况下需要换一个网络环境，比如开手机热点让电脑连上。

## 六、手机热点方案

如果当前网络设备间无法互通，可以用手机开热点：

1. 手机开启**个人热点**
2. 电脑连接手机热点
3. 在电脑上重新查 IP（用 `ipconfig`）
4. 启动前后端
5. 手机用这个 IP 访问（手机流量走 Wi-Fi，也就是热点本身）

注意：手机热点下电脑的 IP 通常是 `192.168.x.x` 格式。

## 七、提醒

- 每次电脑重启或更换网络后，IP 可能会变，需要重新查
- 用完之后记得关掉防火墙规则（如果之前专门开了的话）
- 这个方式只适用于本地开发和测试，正式部署要用服务器
